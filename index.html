CLI Reference

Every command
you need.

wh init
Interactive setup. Provisions S3 bucket + CloudFront automatically. Supports Firebase, Supabase, MongoDB. Optional Edge Worker.
wh ship ⭐ main
Deploy + build + zip in one shot. Encrypts all components, resolves dependency graph, minifies site, writes manifest.
wh deploy
Deploy a single component. Prompts for token and expiry settings.
wh graph-deploy
Deploy all components in a directory and resolve the full dependency graph.
wh atomize new
Split a single HTML page into CDN-powered components automatically. Goes from monolith to micro-frontend in one command.
wh breakdown new
Extract embedded <style> and <script> tags from a single HTML file into separate files.
wh build
Production build — minifies HTML, extracts embedded CSS/JS to content-hashed files.
wh analyze
Detect framework, styling approach, and CDN dependencies automatically from a component directory.
wh convert
Convert vanilla HTML/CSS/JS to React, Vue, Svelte, Next, Angular, or Astro.
wh zip
Zip a directory for deployment to Netlify, S3, cPanel, or any static host.
wh access
Role-based access control for teams. Grant, revoke, and list API keys with scoped permissions.
wh edge-init
Setup Cloudflare Edge Worker — token validation (~5ms), version resolution, geo routing, rate limiting.

Framework Integration

Works in Next.js,
React, Vite — natively.

Reusable React component
"use client";
import { useEffect, useRef } from "react";
import { load } from "webhanger-front";

export default function WHComponent({ name }) {
  const ref = useRef(null);

  useEffect(() => {
    fetch("/wh-manifest.json")
      .then(r => r.json())
      .then(m => {
        const c = m.components[name];
        if (!c || !ref.current) return;
        ref.current.id = `wh-${name}`;
        load(c.urls || c.url, m.pid,
             c.token, c.expires,
             `#wh-${name}`);
      });
  }, [name]);

  return <div ref={ref} />;
}

Security

Encrypted at every
layer of the stack.

🔐
AES-256-GCM Encryption
Every chunk (HTML/CSS/JS) is encrypted before upload. Key derived from SHA-256(projectId + salt). IV + tag + ciphertext stored as base64.
🔏
HMAC-SHA256 Signed URLs
Every request is signed: token = HMAC-SHA256(projectId:path:expires, secretKey). Tamper-proof and project-scoped.
✅
Integrity Check
SHA-256 hash of raw content stored in payload. Verified after decryption to detect any tampering in transit.
🌐
Domain Restriction
Components refuse to load outside specified domains. Configured via allowedDomains option in the SDK.
⏱
Token Expiry
Tokens can be set to never expire, or to expire in a defined number of seconds. Rotate without redeploying via wh.resign().
📄
Manifest-based Delivery
Tokens, projectId, and CDN URLs never appear in HTML. Fetched at runtime from wh-manifest.json.

Token expiry example
// Never expires
wh.deploy("./navbar", "navbar", "1.0.0");

// Expires in 24 hours
wh.deploy("./navbar", "navbar", "1.0.0", {
  expiresInSeconds: 86400
});

// Rotate without re-uploading
await wh.resign("navbar", "1.0.0", {
  expiresInSeconds: 3600
});

Caching

Three-layer cache.
Always fast, always fresh.

Layer 1
localStorage
Components under 50KB are stored inline. Near-instant reads on subsequent loads — no network required.
Layer 2
IndexedDB
Larger components (≥50KB) are stored in IndexedDB for structured persistence without size limits.
Layer 3
Service Worker
Offline + navigation cache via Service Worker. Users get full pages with no network. Stale-while-revalidate throughout.
